Architectural Design Specification: SiliconForge

1. The Chasm in Mixed-Signal Verification

Traditional Mixed-Signal Design Verification (MSDV) relies heavily on discrete-event simulators running Real-Number Models (RNM). While RNMs are fast, they fundamentally fail to model the non-linear distortion, phase noise, and intermodulation distortion inherent to analog components like Phase-Locked Loops (PLLs) and Analog-to-Digital Converters (ADCs).

To work around this, verification engineers often inject arbitrary Gaussian noise into digital models (the "Heuristic Noise Fallacy"). This is mathematically unsound and masks true physical limits. True transient analog simulation via SPICE is mathematically accurate but astronomically slow, making it impossible to use in iterative, continuous integration environments where massive coverage metrics are required.

2. Floquet Theory and Mathematical Determinism

SiliconForge solves this bottleneck not by attempting to speed up analog simulation, but by mathematically mapping the analog behavior to digital constraints.

The core math engine relies on Floquet Theory and the analysis of circuits in Periodic Steady State (PSS). By evaluating the Impulse Sensitivity Function ($\Gamma$), SiliconForge extracts the Perturbation Projection Vector (PPV) from the SPICE output matrices. The PPV definitively calculates how charge injected at different phases of an oscillation causes varying temporal deviations ($\Delta t$).

SiliconForge synthesizes these non-linear phase deviations into deterministic variance equations based on the Root-Mean-Square of the PPV matrix.

3. The Software Architecture: Python Backend to Jinja2 AST

The SiliconForge architecture separates the mathematical extraction from the SystemVerilog generation via an Abstract Syntax Tree (AST) templating approach.

  1. Analog Backend: Python modules interface directly with Ngspice and Xyce. They extract .prn matrices and execute the mathematical derivations described above.
  2. Asset Generator Engine: The processed data (e.g., deterministic jitter bounds, functional coverage ranges) is mapped into Python dictionaries. SiliconForge uses Jinja2 Templating to transform these dictionaries into syntactically correct SystemVerilog and UVM code.
  3. UVM Synthesis: The generator emits raw uvm_sequence_item classes, constrained random bounds (constraint c_phase_noise { ... }), and exhaustive covergroup models that prove the UVM regression stimulated the precise limits dictated by the SPICE physics.

4. Regression Fidelity Analysis

By feeding the generated SystemVerilog constraints back into a standard digital regression environment, SiliconForge demonstrates profound fidelity. In benchmark tests analyzing 9-stage Ring Oscillators (VCO), the jitter histograms generated by the ultra-fast UVM simulations perfectly aligned with the histograms produced by the multi-hour transient SPICE simulations.

This confirms that the generated digital model behaves identically to the physical analog macro, allowing for 100% functional coverage mapping without the runtime penalty of continuous-time solvers.
